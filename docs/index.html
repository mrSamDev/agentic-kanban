agentic-kanban GitHub

Task coordination
for AI agents

SQLite-backed. No server. No daemon. No queues.
Just a shared database file agents use to claim, track, review, and complete work.

Install Commands

Install

curl -sfL https://raw.githubusercontent.com/mrSamDev/agentic-kanban/main/install.sh | sh

Single Go binary + SQLite. Download from releases for production.

Why agentic-kanban

No server

One .db file per project. Agents share it directly. No Redis, Postgres, or message queues.

Atomic claims

Two agents calling claim-next simultaneously get different tasks. SQLite write-serialized.

Crash recovery

15-minute lease per task. log-progress renews it. If an agent crashes, the lease expires and the next claim reclaims the task.

WAL checkpointing

Write-Ahead Log auto-checkpoints every 1000 pages. No unbounded disk growth.

JSON output

Every command prints stable JSON on stdout. Errors go to stderr as {"error":"..."} with exit code 2.

Markdown skills

Role directories contain docs agents read to learn the protocol. No tool-calling protocol needed.

Quick start

# Init a project (creates DB, scaffolds skills)
kanban init --harness pi

# Dispatch work
kanban task dispatch --title "Set up auth" --role worker --priority 10

# Agent claims and completes
kanban task claim-next --agent my-agent --role worker
kanban task log-progress TASK-1 --agent my-agent --note "Working" --type PROGRESS
kanban task complete TASK-1 --agent my-agent

Commands

Command Role Description
task dispatch --title --role [--project] [--priority] any Create a task (status=TODO)
task claim-next --agent --role [--project] worker, reviewer Atomically claim highest-priority task. {} if none.
task log-progress <id> --agent --note [--type] worker Log progress + renew 15-min lease
task block <id> --agent --reason worker Mark blocked, clear lease
task complete <id> --agent [--review] worker Mark done (or submit for review)
task view <id> [--notes] [--history] all Full detail: task + notes + history
task search [--status] [--role] [--agent] [--project] [--limit] manager Filter task list
task approve <id> --agent reviewer Approve IN_REVIEW → DONE
task reject <id> --agent --reason reviewer Reject IN_REVIEW → TODO
task batch set-priority --ids --priority manager Set priority for multiple tasks
task batch set-project --ids --project manager Set project label for multiple tasks
init [--harness] [--plan] [--dir] setup Scaffold DB + skills for pi, claude, or generic

When to use this

Good fit

  • Multiple AI agents need shared state
  • Agents run on the same machine or shared filesystem
  • Durable coordination without Redis / Postgres / queues
  • Crash recovery and task ownership
  • 3–10 concurrent agents (tested up to 50)

Not a fit

  • Agents across untrusted networks
  • Real-time push notifications needed
  • Thousands of concurrent workers
